Implementation plan (engineer handoff)

Visual spec: mockups above. Stack: raw Win32 (windows-rs), no UI framework. Keep system title bar.

Phase 1 Layout & chrome

  1. Remove tall colour ribbon — stop painting full-width green/amber ribbon_rect in src/ui/paint.rs. Replace with a quiet status strip: white background, 4px left accent (#287A28 idle, #00A5FF syncing, red on error). Reuse IDC_SERVER_STATUS static or a new owner-draw child for text + spinner.
  2. Server row — keep IDC_SERVER_HDR + IDC_SERVER_URL_LABEL; add Pair again as owner-draw link (same pattern as IDC_REPO / IDC_AUTHOR in draw.rs), not a grey BS_PUSHBUTTON. Wire to existing do_pair_device. Hide while pairing (same as today’s Re-pair button).
  3. Rename labels in create.rs / messages.rs:
    • “Local backup folder” → “Backup folder on this PC”
    • “Approved folder” → “Server destination”
    • “Download from server” → “Sync from server”
  4. Button order — swap layout: Open then Browse (IDC_OPEN_LOCAL_FOLDER before IDC_BROWSE_LOCAL in layout_window).
  5. Remove Save button — hide/remove IDC_SAVE from create.rs and layout_window. Reclaim width in bottom bar.
  6. Footer sync strip — place above checkboxes:
    • Idle: static “Ready” (or hide strip).
    • Busy: text {done}/{total} · {pct}% + optional ETA (logic already in on_app_sync_activity).
    • Wire mkprogress → IDC_SYNC_PROGRESS (msctls_progress32); update via PBM_SETPOS in on_app_sync_activity when progress.1 > 0.
  7. Window height — update calc_window_height / layout_window in commands.rs: subtract RIBBON_H, add status-strip + sync-footer heights; adjust WndState layout fields in common.rs.

Phase 2 Auto-save (replaces Save)

Extract do_save body in commands.rs into persist_settings(hwnd) -> Result<(), String> (read ctrls, validate, lock paired fields, encrypt, config::save, apply_startup, restart_sync_engine).

Update README “When the engine starts” table: remove Save row; add Browse + checkbox triggers.

Phase 3 Inline row progress (hard)

Standard LISTBOX is single-line strings today (WM_APP_LOG → LB_INSERTSTRING). Mock shows two-line rows with mini bars — requires one of:

  • Option A (recommended): Replace activity list with owner-draw list (LBS_OWNERDRAWFIXED) or a child panel that paints rows in WM_DRAWITEM / WM_PAINT. Store Vec<ActivityRow> in WndState (icon, label, optional pct 0–100, done flag).
  • Option B: Keep listbox for text; draw mini bars in parent client area under list — fragile on resize/scroll.

Data for per-row %: not emitted today. sync.rs logs Uploading: path but ActivityInfo only has batch completed/total. For real per-file bars:

  • Extend upload path to report bytes sent / file size (or stepped 0/50/100).
  • Post per-file updates on WM_APP_LOG or new WM_APP_FILE_PROGRESS.
  • Until then: indeterminate mini bar (animated) on rows matching activity_replaces “Uploading …” — still improves UX vs text-only.

Phase 4 Status copy & tray

  • Quiet strip text mirrors ribbon rules in update_ribbon_after_sync (utils.rs) — shorten for strip width (“All synced”, “Syncing · 3 left”).
  • Tray tooltip can keep detailed {done}/{total} · ETA · {pct}% from on_app_sync_activity.
  • Remove duplicate % from old ribbon-only placement once footer bar exists.

Acceptance criteria

  • Paired window matches mockup structure (idle + syncing screenshots).
  • No Save button; folder + checkbox changes persist and restart sync without extra click.
  • Batch progress visible as footer bar + fraction/ETA text while syncing.
  • Activity rows show mini progress for in-flight uploads (determinate when data available, else indeterminate).
  • Pair again works from link; paired server/destination fields stay locked.
  • cargo build --release clean; manual test: pair, browse, toggle sync-from-server, watch footer bar + list during upload.

Out of scope

  • Custom borderless title bar (keep system caption).
  • egui / webview / third-party UI.
